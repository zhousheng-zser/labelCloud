# KITTI Dataset 标定文件 (calib.txt) 解析

在 KITTI 数据集中，`calib.txt` 记录了各个传感器（激光雷达、多个摄像头、IMU惯导系统）之间的空间几何关系和相机的内参。它的主要作用是**将不同坐标系下的数据进行投影对齐**（例如将 3D 点云投影到 2D 图像平面，或者将 3D 目标框在不同的相机和雷达坐标系间转换）。

对于目标检测任务（尤其是 3D 检测），最重要的参数是 `P2`、`R0_rect` 和 `Tr_velo_to_cam`。

---

## 逐行参数含义

### 1. `P0`, `P1`, `P2`, `P3` (相机投影矩阵)
这四行分别是四个不同摄像头的投影矩阵 (Projection Matrix)。
- **P0**: 左侧灰度相机的投影矩阵
- **P1**: 右侧灰度相机的投影矩阵
- **P2**: 左侧彩色相机的投影矩阵 **（最常用，KITTI 的 2D 图像和 3D Bbox 标准都是基于 P2 相机）**
- **P3**: 右侧彩色相机的投影矩阵

**数据格式**:
每个投影矩阵的大小为 `3x4`，展开成 12 个数值。
包含了**相机的内参**（焦距、光心）以及对应 0 号相机（P0）的**基线平移**（外参的一部分）。

**作用**:
用于将“矫正后的 3D 相机坐标系 (Rectified Camera Frame)”中的点 `(x, y, z)`，投影到 2D 图像的像素平面坐标 `(u, v)` 上。公式：`Y_image = P2 * X_rect`。

### 2. `R0_rect` (相机矫正旋转矩阵)
**数据格式**:
大小为 `3x3`，展开成 9 个数值。通常在文件中以 1x9 的数组记录。

**作用**:
在多相机系统中，为了进行立体匹配或标准化各个相机的朝向，需要进行立体矫正。`R0_rect` 表示将 0 号相机（参考相机）的几何坐标系旋转到一个共面的“矫正后坐标系 (Rectified Frame)”中。
在做雷达到相机的转换时，任何转换到相机坐标的点，都要乘上这个矩阵，以对其进行校正。

### 3. `Tr_velo_to_cam` (雷达到相机的外参变换矩阵)
**数据格式**:
大小为 `3x4` 的变换矩阵 `[R | T]`，展开成 12 个数值。
- **左边 3x3 (R)**: 激光雷达 (Velodyne) 到相机的**旋转**矩阵。
- **右边 3x1 (T)**: 激光雷达 (Velodyne) 到相机的**平移**向量。

**物理意义**:
记录了雷达传感器和参考相机之间的物理安装位置差异。
- 原版 KITTI 数据中，雷达安装在相机后方和上方，所以存在如 `-0.27m` 的 Z 轴平移等真实无物理偏差。
- **我们修改为无平移零误差版本的原因**：当你自建纯 3D 点云数据集而没有物理相机时，可以直接将 `T` 设为 `0`，保留 `R` 作为单纯的坐标轴朝向转换（如：将雷达的 X轴(前) 映射到 相机的 Z轴(前) 等），这样可以使 labelCloud 导出的包围框完美对齐点云，不会在可视化时飘在空中。

### 4. `Tr_imu_to_velo` (IMU 到雷达的外参变换矩阵)
**数据格式**:
大小为 `3x4`，展开成 12 个数值，格式同样是 `[R | T]`。

**作用**:
将 GPS/IMU 惯性导航系统的坐标系转换到激光雷达的坐标系中。在纯障碍物检测任务中，如果我们不需要使用时间序列上的运动补偿、GPS信息或里程计同步，通常可以忽略此参数。

---

## 点云坐标如何转换到图像？

一个从激光雷达观测到的 3D 点 `X_velo = (x, y, z, 1)`，如果要转换到 P2 相机所在的 2D 照片上的某一个像素位置 `Y_image = (u, v, 1)`，其严格的级联乘法转换公式为：

**`Y_image = P2 * R0_rect * Tr_velo_to_cam * X_velo`**

这几行短短的文本，构成了整个自动驾驶多传感器融合算法中空间几何对齐的基石。
